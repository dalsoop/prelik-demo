⚙️ dalforge

하나의 dal이
모든 것을 담는다

dalforge의 핵심은 dal이라는 단위입니다. CLI 도구, 스킬 파일, 훅 스크립트, AI 플레이어, 시크릿 볼트가 각각 고유한 DAL ID를 가지고 하나의 컨테이너에서 동작합니다.

  • ⚡
    DAL:CLI
    dalcenter join 시 발급. 클라우드 허브의 CLI 패키지도 관리.
  • 🧩
    DAL:SKILL
    .dalfactory/skills/ → ~/.claude/skills/ 심링크 자동 export.
  • 🪝
    DAL:HOOK
    훅 스크립트 → settings.json 자동 머지. AI 런타임에 즉시 반영.
  • 🤖
    DAL:PLAYER
    claude_sdk / codex_appserver / gemini_cli — LXC 내부 실행 에이전트.
backend.cue — DalTemplate
package dalfactory backend: #DalTemplate & { // ── Container ───────────────── container: { image: "ubuntu-24.04" memory: 2048 cpus: 2 } // ── Skills ──────────────────── skills: [ "skills/code-review.md", "skills/deploy.md", ] // ── Hooks ───────────────────── hooks: [ "hooks/pre-commit.sh", ] // ── Exports ─────────────────── exports: { claude_sdk: { skills: skills hooks: hooks } } // ── Secrets ─────────────────── secrets: [ "GITHUB_TOKEN", "DATABASE_URL", ] }
🔐 VeilKey

시크릿은 절대
평문으로 남지 않는다

VeilKey는 AES-256-GCM으로 암호화된 SQLite에 자격증명을 저장합니다. dal 프로비전 시 자동으로 환경변수에 주입되어, 개발자는 시크릿을 직접 다룰 필요가 없습니다.

  • 🔒
    AES-256-GCM 암호화
    모든 자격증명은 저장 즉시 암호화. 복호화는 런타임에만.
  • 💉
    자동 환경변수 주입
    dalcenter provision 시 컨테이너에 env로 자동 주입.
  • 🏰
    프로젝트별 격리
    각 dal은 독립된 Vault를 가집니다. 교차 접근 불가.

01 / Problem
AI가 여기저기 흩어지는 문제
Before vs After dalforge
문
제
레포마다 AI 설정이 달라지고, 시크릿이 노출되고, 에이전트가 꺼져도 모른다.
✕  기존 방식
  • ✕AI 설정이 레포마다 제각각 — 일관성 없음
  • ✕어느 레포에 어떤 에이전트가 붙었는지 파악 불가
  • ✕.env 파일 평문 시크릿 — 레포 노출 시 전부 유출
  • ✕에이전트 다운 감지 안 됨 — 수동 재기동
  • ✕스킬, 훅, 설정 레포마다 중복 복붙
  • ✕AI 전체 현황 한눈에 불가 — 분산되어 있음
✓  dalforge 방식
  • ✓.dalfactory/dal.cue 하나로 AI 환경 전체 선언
  • ✓dalcenter list — 전체 달 상태 한 명령으로 조회
  • ✓AES-256-GCM SecretVault — 암호화 저장, 자동 주입
  • ✓watch 60s 자동 감시 → drift 감지 즉시 reconcile
  • ✓스킬/훅 .dalfactory 관리 → .claude/, .codex/ 자동 export
  • ✓레지스트리 기반 전체 관리 — DAL ID로 완전 추적
02 / How It Works
4단계로 시작하는 레포 AI
Join → Provision → Watch → Done
4
STEPS.
DONE.
1
Step 01
레포에 선언
레포 루트에 .dalfactory/dal.cue를 생성합니다. 사용할 AI PLAYER, 스킬, 훅, 시크릿을 CUE 문법으로 선언. dalcenter validate로 즉시 검증.
$ dalcenter validate
2
Step 02
달 등록
dalcenter join 실행 — 레포가 달포지 레지스트리에 연결됩니다. 고유 DAL:CLI:uuid8 ID 발급. SQLite 레지스트리에 영구 기록.
$ dalcenter join
3
Step 03
컨테이너 기동
Proxmox LXC 컨테이너가 자동 프로비저닝됩니다. Ubuntu 24.04 베이스, AI PLAYER 설치, SecretVault에서 시크릿 자동 복호화 주입.
$ dalcenter provision <id>
4
Step 04
자동 감시
watch 모드 — 60초마다 전체 달 상태 점검. 컨테이너 drift 감지 시 자동 reconcile 실행. 에이전트가 꺼지면 스스로 재기동합니다.
$ dalcenter watch

.dalfactory/dal.cue
레포의 AI 환경을 선언하는
단일 소스 파일.
CUE 언어로 작성.
.dalfactory/dal.cue
package dalfactory

import "dalforge.io/schema"

dal: schema.Dal & {
  // AI 에이전트 플레이어 지정
  player:  "claude"

  // 스킬 목록 (cloud hub에서 설치)
  skills: ["code-review", "pr-check", "test-gen"]

  // 훅 이벤트
  hooks: ["pre-commit", "post-push"]

  // 시크릿 (SecretVault에서 자동 주입)
  secrets: ["GITHUB_TOKEN", "OPENAI_KEY"]

  // LXC 컨테이너 설정
  container: {
    base:   "ubuntu-24.04"
    memory: 2048
    cores:  2
  }
}

01 — The Problem
AI가 여기저기
흩어져 있다면.
BEFORE — 기존 방식
  • ✕ AI 설정이 레포마다 제각각이라 누가 어디에 뭘 붙였는지 파악이 안 됨
  • ✕ 시크릿이 .env 파일에 평문으로 존재 — 노출 위험
  • ✕ 에이전트 죽어도 모름, 재시작도 수동, 업데이트도 수동
  • ✕ 스킬, 훅, 환경변수 관리 분산 — 레포마다 중복 설정
  • ✕ AI 사용 현황 전체 파악 불가능 — 한눈에 보이지 않음
→
AFTER — dalforge 방식
  • ✓ .dalfactory/dal.cue 하나로 레포 AI 환경 전체 선언 — SSOT
  • ✓ AES-256-GCM SecretVault — 암호화 저장, 컨테이너 기동 시 자동 주입
  • ✓ watch 모드 60s 자동 감시 — drift 감지 즉시 reconcile 자동 실행
  • ✓ 스킬/훅 .dalfactory에서 관리 → .claude/, .codex/ 자동 export
  • ✓ dalcenter list — 전체 달의 상태를 한 명령으로 조회
02
02 — How It Works
4 STEPS.
DONE.
step 01
레포에 선언
레포 루트에 .dalfactory/dal.cue 파일을 생성하고 사용할 AI 에이전트, 스킬, 훅, 시크릿을 CUE 문법으로 선언합니다. dalcenter validate 로 유효성 즉시 검사.
$ dalcenter validate
step 02
달 등록
dalcenter join 실행 시 레포가 달포지 레지스트리에 연결됩니다. 고유한 DAL ID (DAL:CLI:uuid8) 가 발급되고 SQLite 레지스트리에 기록됩니다.
$ dalcenter join
step 03
컨테이너 기동
Proxmox LXC 컨테이너가 자동 프로비저닝됩니다. Ubuntu 24.04 베이스, AI 에이전트 설치, SecretVault 자동 주입. PLAYER (claude/codex/gemini) 즉시 실행.
$ dalcenter provision <id>
step 04
자동 감시
watch 모드를 켜면 60초마다 전체 달의 상태를 점검합니다. 컨테이너 drift 감지 시 자동 reconcile. 멈추면 스스로 재기동합니다.
$ dalcenter watch

dalcenter CLI는 15개의 서브커맨드로 AI 에이전트 컨테이너의 전체 생애주기를 관리합니다. Join → Provision → Watch → Destroy.

dal.cue 유효성 검사
운영 명령
provision
LXC 컨테이너 프로비저닝
watch
60s 자동 감시 루프
reconcile
drift 감지 및 자동 복구
destroy
컨테이너 완전 제거
보안 / 내보내기
secret
AES-256-GCM 시크릿 관리
export
스킬/훅 .claude/ 내보내기
catalog

start / stop

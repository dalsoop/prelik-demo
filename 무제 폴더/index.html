Chapter I

인형이
깨어난다

dal은 단순한 프로세스가 아니다.
이름이 있고, 기억이 있고,
고유한 영혼(ID)이 있다.

컨테이너 안에 claude, codex, gemini가 이미 로그인된 상태로 존재한다. 하나의 dal은 하나의 작업 환경이다. 이름이 바뀌어도, 재시작해도 — ID는 영원히 고정된다.

DAL:CONTAINER:a1b2c3d4
my-agent
ALIVE
Chapter II

안에서
말하는 자들

PLAYER는 dal 안에서 실제로 일하는 주체다. 하나의 dal에 여러 PLAYER가 공존할 수 있다. 각자 다른 목소리로, 다른 도구로, 같은 목표를 향해 움직인다.

✦
Claude
PLAYER · 창조자
설계하고, 작성하고, 판단한다. 코드와 문서와 결정이 claude의 언어로 흐른다.
◈
Codex
PLAYER · 실행자
정밀하게 구현하고, 반복 작업을 처리한다. 끊임없는 코드의 흐름을 만든다.
⬡
Gemini
PLAYER · 관찰자
분석하고, 검토하고, 다른 시각을 더한다. 시스템 전체를 바라보는 눈이다.
Chapter III

소환의
의식

I
선언
레포 안에 .dalfactory/dal.cue를 만든다. 어떤 컨테이너 위에서, 어떤 에이전트가, 어떤 스킬을 갖고 살아갈지 CUE로 선언한다.
II
소환 — join
dalcenter가 선언을 읽고 영혼에 ID를 부여한다. 스킬과 훅이 .claude/로 export된다.
dalcenter join /path/to/repo
III
형체 부여 — provision
Proxmox LXC 위에 컨테이너를 생성한다. 패키지가 설치되고, 에이전트가 깨어난다.
dalcenter provision my-dal --vmid 219
IV
감시 — watch
선언과 현실 사이의 drift를 끊임없이 감지하고 수복한다. 영혼은 선언한 모습으로 존재한다.
dalcenter watch --interval 60
∞
소멸 — dismiss
일이 끝나면 dal은 사라진다. unexport, destroy. 그러나 ID는 기록에 남는다.
dal dismiss my-agent
.dalfactory/dal.cue
// 이 레포의 dal 선언
name: "my-agent" version: "1.0.0" template: "claude-dev"
container: { base: "ubuntu-24.04" packages: [ "bash", "python3", "tmux" ] }
agents: { claude: { enable: true } codex: { enable: true } }
export: { skills: ["agent-coach"] hooks: ["on-stop"] }
build: { entry: "./run.sh" health_check: "./check.sh" }

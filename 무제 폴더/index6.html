How it works
.dalfactory 하나.
나머지는 dalcenter.
선언하면 dalcenter가 등록하고, 컨테이너를 만들고, 에이전트를 설치하고, 60초마다 상태를 점검한다.
01 · DECLARE
선언
레포 안에 .dalfactory/dal.cue를 만든다. 어떤 AI가, 어떤 환경에서 살지 CUE로 선언.
name: "my-agent"
agents: { claude: true }
02 · JOIN
등록
dalcenter join이 manifest를 읽고 인스턴스를 등록한다. 영구 ID 발급, skills export.
$ dalcenter join .
✓ DAL:CONTAINER:a1b2
03 · PROVISION
배포
Proxmox LXC 컨테이너 생성. 패키지 설치, API 키 주입, 에이전트 시작.
$ dalcenter provision
✓ provisioned (vmid=219)
04 · WATCH
감시
60초마다 선언과 현실의 drift를 감지하고 자동 수복. 에이전트는 영원히 선언한 상태.
$ dalcenter watch
✓ all instances: ok

01 · Join
레포를 등록하면 AI가 붙는다
.dalfactory/dal.cue를 읽고 인스턴스를 레지스트리에 등록한다. 영구 ID가 발급되고, Claude 스킬과 훅이 .claude/로 export된다.
SHELL
# manifest 검증 후 등록
$ dalcenter validate . && dalcenter join .
instance created: DAL:CONTAINER:a1b2c3d4
template=claude-dev skills=3 hooks=1
💡
클라우드 패키지도 동일하다. dalcenter join agent-coach — dalforge 허브에서 자동으로 가져온다.
02 · Provision
컨테이너가 생성되고 AI가 깨어난다
Proxmox LXC를 자동 생성하고 packages, agents를 설치한다. API 키는 SecretVault에서 자동 주입된다. --dry-run으로 사전 확인 가능.
SHELL
$ dalcenter provision my-agent --vmid 219
✓ ubuntu-24.04 container created
✓ claude installed · API keys injected
✓ provisioned (vmid=219)
💡
--dry-run으로 실행 전 명령어를 미리 확인할 수 있다.
03 · Watch
선언이 현실이 된다, 계속
watch 데몬이 interval마다 모든 인스턴스를 점검한다. 선언과 실제 상태 사이의 drift를 발견하면 자동으로 수복한다.
SHELL
$ dalcenter watch --interval 60
watch started (interval=60s, pid=48291)
DAL:CONTAINER:a1b2c3d4: ok
DAL:CONTAINER:7e4b9d15: ok
DAL:CONTAINER:c9d0e1f2: repaired
— skill agent-coach: re-exported
💡
dalcenter reconcile로 즉시 1회 점검도 가능하다. 싱글톤 PID 락으로 watch는 하나만 실행된다.
04 · SecretVault
키는 자동으로 주입된다
API 키를 AES-256-GCM으로 암호화해서 SQLite에 보관한다. 컨테이너 배포 시 자동 복호화·주입. 사람 손을 거치지 않는다.
SHELL
$ dalcenter secret set anthropic_api_key
secret "anthropic_api_key" saved

$ dalcenter secret list
NAME UPDATED
anthropic_api_key 2026-03-17
openai_api_key 2026-03-17
💡
.dalfactory의 secret_keys에 이름을 선언하면 provision 시 자동으로 컨테이너에 주입된다.
05 · Destroy
필요없으면 제거한다
컨테이너를 중지하고 제거한다. 레지스트리 기록은 남는다. ID는 영원히 고정이다. dalcenter join → provision으로 즉시 복원 가능.
SHELL
# dry-run으로 먼저 확인
$ dalcenter destroy my-agent --dry-run
dry-run: pct stop 219
dry-run: pct destroy 219

$ dalcenter destroy my-agent
destroyed
💡
ID는 레지스트리에 영구 보존된다. 언제든 dalcenter join → provision으로 동일한 에이전트를 복원할 수 있다.

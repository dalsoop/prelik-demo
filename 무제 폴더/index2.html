Features

핵심 기능

01
📦
클라우드 패키지 허브
npm registry처럼 dalcli 패키지를 검색하고 이름 하나로 원격 등록한다. catalog search, join agent-coach.
02
🔄
Reconcile / Watch
선언과 현실 간의 drift를 자동 감지하고 수복한다. watch 데몬이 60초마다 전체 인스턴스를 점검한다.
03
🔐
SecretVault
API 키를 AES-256-GCM으로 암호화 보관. 컨테이너 시작 시 자동 복호화·주입. 인간의 손을 거치지 않는다.
04
🖥️
Proxmox LXC 프로비저닝
Ubuntu 24.04 LXC를 자동 생성하고 패키지·에이전트를 설치한다. --dry-run으로 사전 검증 가능.
05
📤
Skills / Hooks Export
.dalfactory 선언에서 Claude 스킬과 훅을 .claude/로 자동 export. Codex skills도 지원.
06
🪪
영구 ID 체계
DAL:CATEGORY:uuid8. 이름이 바뀌어도 재시작해도 ID는 고정. CLI·PLAYER·CONTAINER·SKILL·HOOK.

Quick Start

5분 안에
시작하기

1
빌드 & 설치
Go 1.25+ 환경에서 dalcenter를 빌드한다.
SHELL
# 레포 클론
$ git clone https://github.com/dalsoop/dalforge
$ cd dalforge

# 빌드
$ go build -o ~/go/bin/dalcenter ./cmd/dalcenter/
✓ installed → ~/go/bin/dalcenter
2
.dalfactory 선언
레포 루트에 .dalfactory/dal.cue를 만든다.
CUE
name: "my-agent"
version: "1.0.0"
template: "claude-dev"

container: {
  base: "ubuntu-24.04"
  packages: ["bash", "tmux"]
}

agents: { claude: { enable: true } }
3
등록 & Export
dalcenter가 manifest를 읽고 인스턴스를 등록한다.
SHELL
$ dalcenter validate .
ok .dalfactory/dal.cue

$ dalcenter join .
instance created: DAL:CONTAINER:a1b2c3d4
template=claude-dev skills=3
4
Proxmox 프로비저닝
LXC 컨테이너를 생성하고 에이전트를 설치한다.
SHELL
$ dalcenter provision my-agent \
  --vmid 219 --storage local-lvm

provisioned (vmid=219)
5
Watch 루프 시작
60초마다 전체 인스턴스 상태를 점검하고 drift를 수복한다.
SHELL
$ dalcenter watch --interval 60
watch started (interval=60s, pid=48291)
  DAL:CONTAINER:a1b2c3d4: ok
.dalfactory/dal.cue
// 레포 선언 SSOT
name: "my-agent"
version: "1.0.0"
template: "claude-dev"

container: {
  base: "ubuntu-24.04"
  packages: [
    "bash", "python3", "tmux"
  ]
}

agents: {
  claude: { enable: true }
  codex: { enable: true }
}

export: {
  skills: ["agent-coach"]
  hooks: ["on-stop"]
}

build: {
  entry: "./run.sh"
  health_check: "./check.sh"
}

secret_keys: [
  "anthropic_api_key",
  "openai_api_key"
]

How it works
세 단계로 끝난다
01 · DECLARE
📋
선언
레포 안에 .dalfactory/dal.cue를 만든다. 어떤 에이전트가, 어떤 컨테이너에서, 어떤 스킬로 살아갈지 CUE로 선언한다.
name: "pr-reviewer"
agents: { claude: { enable: true } }
container: { base: "ubuntu-24.04" }
02 · SUMMON
⚡
소환
dalcenter join으로 인스턴스를 등록하고, provision으로 Proxmox LXC 컨테이너를 생성한다. 에이전트가 깨어난다.
✓ DAL:CONTAINER:3a8c1f02
✓ claude installed
✓ provisioned (vmid=219)
03 · OPERATE
🔄
운영
watch 데몬이 60초마다 선언과 현실의 drift를 감지하고 수복한다. 에이전트는 선언한 모습으로 영원히 존재한다.
watch started (pid=48291)
✓ DAL:CONTAINER:3a8c1f02: ok
reconcile: clean
Features
달포지가 하는 일
🎯
영구 ID — 이름이 바뀌어도 살아있다
모든 에이전트는 DAL:CONTAINER:uuid8 ID를 가진다. 재시작해도, 이름을 바꿔도 ID는 고정이다. 에이전트의 정체성이 영속한다.
DAL:CONTAINER:a1b2c3d4
📦
클라우드 패키지 허브
npm처럼 dalcli 패키지를 이름 하나로 검색하고 설치한다. agent-coach, task-queue, lxc-stage-player 등 즉시 사용 가능.
dalcenter catalog search
🔐
SecretVault — 키가 자동 주입된다
API 키는 AES-256-GCM으로 암호화되어 보관된다. 에이전트가 깨어날 때 자동으로 복호화하여 주입한다. 인간의 손을 거치지 않는다.
AES-256-GCM · SQLite
🖥️
Proxmox LXC 프로비저닝
ubuntu-24.04 컨테이너를 자동 생성하고 에이전트를 설치한다. --dry-run으로 사전 검증, --vmid로 VMID 직접 지정.
dalcenter provision
⚖️
Reconcile — 선언이 현실이 된다
dalcenter watch가 60초마다 .dalfactory 선언과 실제 상태를 비교한다. 불일치를 자동 수복한다. drift는 용납되지 않는다.
dalcenter watch --interval 60
📤
Skills / Hooks Export
.dalfactory 선언에서 Claude 스킬과 훅을 .claude/로, Codex 스킬을 .codex/로 자동 export한다. 에이전트 설정이 코드로 관리된다.
dalcenter export
Examples
이런 에이전트를 만들 수 있다
"PR 리뷰하는 Claude 에이전트 배포해줘"
∨
.dalfactory/dal.cue
name: "pr-reviewer"
template: "claude-dev"
agents: { claude: { enable: true } }
container: { base: "ubuntu-24.04" }
export: { skills: ["pr-review"] }
실행 순서
$ dalcenter validate .
$ dalcenter join .
$ dalcenter provision pr-reviewer --vmid 220
$ dalcenter watch
✓ DAL:CONTAINER:3a8c1f02 · ok
"claude랑 codex 둘 다 쓰는 풀스택 에이전트"
∨
.dalfactory/dal.cue
name: "full-stack-agent"
agents: {
  claude: { enable: true }
  codex: { enable: true }
}
export: { skills: ["agent-coach"] }
실행 순서
$ dalcenter join .
$ dalcenter provision full-stack-agent
✓ claude installed
✓ codex installed
✓ provisioned (vmid=221)
"agent-coach 패키지를 클라우드에서 바로 가져와서 실행해줘"
∨
로컬 파일 불필요
# 클라우드에서 직접 가져옴
$ dalcenter join agent-coach

# dalforge hub에서 staged
staged: dalcli-agent-coach → ~/.dalcenter/sources/
실행 순서
$ dalcenter catalog search agent
$ dalcenter join agent-coach
✓ staged from dalforge cloud
✓ DAL:CLI:3a8c1f02 created
$ dalcenter start agent-coach
"API 키 안전하게 저장하고 에이전트에 주입해줘"
∨
SecretVault 설정
# dal.cue에 키 이름만 선언
secret_keys: [
  "anthropic_api_key",
  "openai_api_key"
]
실행 순서
$ dalcenter secret set anthropic_api_key
✓ AES-256-GCM 암호화 저장
$ dalcenter provision my-agent
✓ API keys injected
에이전트가 키를 받아 실행 시작
Quick Start
5분 안에 시작하기
01
빌드
Go 1.25+ 환경에서 dalcenter를 빌드한다.
SHELL
$ git clone https://github.com/dalsoop/dalforge && cd dalforge
$ go build -o ~/go/bin/dalcenter ./cmd/dalcenter/
✓ ~/go/bin/dalcenter
02
.dalfactory 선언
레포 루트에 선언 파일을 만든다. 이게 전부다.
CUE — .dalfactory/dal.cue
name: "my-agent" // 에이전트 이름
template: "claude-dev"
container: { base: "ubuntu-24.04" }
agents: { claude: { enable: true } }
03
등록 & 소환
dalcenter가 선언을 읽고, 에이전트에 영구 ID를 부여한다.
SHELL
$ dalcenter validate . && dalcenter join .
instance created: DAL:CONTAINER:a1b2c3d4
04
Proxmox에 배포
LXC 컨테이너를 생성하고 에이전트가 깨어난다.
SHELL
$ dalcenter provision my-agent --vmid 219
✓ provisioned (vmid=219)
✓ claude installed · API keys injected
05
영원히 살게 한다
watch가 drift를 감지하고 에이전트를 선언한 상태로 유지한다.
SHELL
$ dalcenter watch --interval 60
watch started (interval=60s, pid=48291)
DAL:CONTAINER:a1b2c3d4: ok

Reconcile Cycle
각 레포는 하나의 세계다.
그 세계에는 전담 AI가 있어야 한다.

지금까지 AI 에이전트는 서비스로 존재했다. 하나의 AI를 여러 프로젝트가 공유하고, 설정은 분산되었으며, 시크릿은 어딘가 숨겨져 있었다. 어느 레포에 어떤 AI가 붙어 있는지 아무도 정확히 알지 못했다.

달포지는 이 패러다임을 뒤집는다. 레포마다 독립된 AI 에이전트 컨테이너가 프로비저닝된다. 선언하고, 등록하고, 기동하면 — 감시와 복구가 자동으로 이루어진다. 레포가 곧 AI의 경계다.

How It Works
01
레포에 선언하다
.dalfactory/dal.cue 파일 하나로 레포의 AI 환경 전체를 선언합니다. 사용할 PLAYER (claude, codex, gemini), 스킬, 훅, 시크릿 — 모두 CUE 문법으로. 이것이 레포 AI의 단일 소스입니다.
$ dalcenter validate
02
달로 등록하다
dalcenter join — 레포가 달포지 레지스트리에 연결됩니다. 고유한 DAL:CLI:uuid8 ID가 발급되고, SQLite 레지스트리에 영구 기록됩니다. 이제 이 레포는 달포지가 아는 세계의 일부입니다.
$ dalcenter join
03
컨테이너를 기동하다
Proxmox LXC 컨테이너가 자동으로 프로비저닝됩니다. Ubuntu 24.04 베이스, AI 에이전트 설치, SecretVault에서 AES-256-GCM 복호화 후 환경변수 자동 주입. PLAYER가 컨테이너 안에서 깨어납니다.
$ dalcenter provision <id>
04
스스로 감시하다
watch 모드를 켜면 60초마다 등록된 모든 달의 상태를 점검합니다. 컨테이너 drift가 감지되면 즉시 reconcile이 실행됩니다. 에이전트는 꺼지지 않습니다 — 꺼지면 스스로 켜집니다.
$ dalcenter watch

단 하나의 선언 파일.
.dalfactory/dal.cue는 레포 AI 환경의 유일한 진실 공급원입니다. PLAYER, 스킬, 훅, 시크릿, 컨테이너 설정 — 모두 이 파일 하나에서 선언됩니다. CUE의 타입 시스템이 유효성을 보장합니다.
.dalfactory/dal.cue
package dalfactory

import "dalforge.io/schema"

dal: schema.Dal & {
  // AI 플레이어 지정
  player: "claude"

  // 스킬 (cloud hub)
  skills: ["code-review", "pr-check"]

  // SecretVault에서 자동 주입
  secrets: ["GITHUB_TOKEN", "OPENAI_KEY"]

  // LXC 컨테이너 설정
  container: {
    base:   "ubuntu-24.04"
    memory: 2048
    cores:  2
  }
}

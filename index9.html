Plaintext secrets are the vulnerability you forgot to patch.

VeilKey intercepts every secret at the moment of creation and replaces it with an encrypted, scoped reference. The original plaintext is destroyed. What remains — VK:LOCAL:a1b2c3d4 — is visible, auditable, and permanently unreadable without the key hierarchy.

Workflow

Four operations. Zero plaintext.

01

Scan

222+ patterns detect AWS keys, GitHub tokens, private keys, JWTs, database URLs. Runs fully offline.

vk scan .env
02

Filter

Replace every detected secret with an AES-256-GCM encrypted VK: reference. Plaintext destroyed.

vk filter .env
03

Wrap

Execute commands with output masking. If a secret leaks to stdout, it's caught and replaced.

vk wrap ./deploy.sh
04

Exec

Resolve VK: tokens at runtime. Decrypted in process memory only — never disk, never env vars.

vk exec ./app

AI Safety

The agent never touches real credentials.

AI agents work with VK: references. Real secrets exist only in process memory at exec time.

  • AI reads VK:LOCAL:xxxx — works normally
  • Real secret injected at exec, never on disk
  • Zero exposure in chat, logs, or git
  • No workflow changes required

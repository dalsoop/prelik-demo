Core Concept

Secret은 Memory에만 존재합니다.
Disk에는 절대 저장되지 않습니다.

Secret이 VeilKey에 입력되는 순간, AES-256-GCM으로 암호화되고 범위 지정된 Reference Token으로 대체됩니다. Config 파일, Git History, CI Log — 어디에도 Plaintext는 존재하지 않습니다.

  • AI Agent는 VK: Reference만 봅니다 — 정상 작동, 위험 제로
  • 재시작 없이 Node별 Key 교체 가능
  • 각 Node는 독립된 DEK 보유 — 측면 이동 공격 원천 차단
  • Password는 파일에서만 읽습니다 — 환경 변수 사용 불가
Architecture

Hub-and-Spoke 구조. 모든 Node는 완전히 격리됩니다.

KeyCenter 하나가 Policy를 결정합니다. LocalVault Node는 Ciphertext만 보관합니다. Spoke가 침해되더라도 — Hub와 다른 모든 Spoke는 안전하게 봉인됩니다.

KC

KeyCenter

:10181

Control Plane. Node 식별, DEK Lifecycle, Rotation Policy, Secret Registry 관리. 배포당 단일 Instance.

LV

LocalVault

:10180

Node Agent. Ciphertext 저장, 5분마다 Heartbeat. KeyCenter 없이는 복호화 불가. 머신당 하나.

PX

Proxy

:18080

Egress 강제 적용. Outbound HTTP 가로채기, Plaintext 유출 차단, VK:TEMP: Token 발급. 선택 사항.

CLI

CLI

 

운영자 도구. scan, filter, wrap, exec, resolve. 자동 Secret Masking이 적용된 PTY Wrapping.

Features

AI 시대를 위해 설계되었습니다.

AI-Safe Reference

AI Agent는 VK: Token으로만 작동합니다. 실제 Credential을 보지 않고도 읽고, 쓰고, 실행합니다.

Zero Plaintext at Rest

입력 시점에 AES-256-GCM Encryption. Config 파일, Git History, Log — 어디에도 Plaintext 없음.

무중단 Key Rotation

KeyCenter가 Scheduling, 각 Node가 재암호화. 재시작 없음, Downtime 없음, 조율 불필요.

222가지 이상의 Pattern

AWS, GCP, GitHub, Stripe, Private Key, JWT 감지. 완전 Offline Scan — API 불필요.

Egress 강제 적용

선택적 Proxy가 Outbound HTTP의 Plaintext를 차단합니다. Veilroot Shell Guard로 직접적인 curl/wget 차단.

Self-Hosted

SQLite Backend, Cloud 의존성 없음. macOS launchd 또는 Linux systemd. AGPL-3.0 License.

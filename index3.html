The Idea

Secrets exist in memory.
Never on disk.

The moment a secret enters VeilKey, it's encrypted with AES-256-GCM and replaced with a scoped reference token. Your config files, your git history, your CI logs — none of them ever contain plaintext.

  • AI agents see only VK: references — work normally, zero risk
  • Rotation happens per-node without restarts
  • Each node has its own DEK — lateral movement is dead
  • Password read from file, never environment variable
Architecture

Hub-and-spoke. Every node isolated.

One KeyCenter decides policy. LocalVault nodes hold only ciphertext. Compromise a spoke — the hub and every other spoke stays sealed.

Features

Built for the AI era.

AI-safe references

AI agents work with VK: tokens. They read, write, and execute — without ever seeing real credentials.

Zero plaintext at rest

AES-256-GCM encryption at point of entry. Configs, git history, and logs contain only references.

Seamless rotation

KeyCenter schedules, each node re-encrypts. No restart, no downtime, no coordination needed.

222+ patterns

Detect AWS, GCP, GitHub, Stripe, private keys, JWTs. Fully offline scan — no API required.

Egress enforcement

Optional proxy blocks plaintext in outbound HTTP. Veilroot shell guards block direct curl/wget.

Self-hosted

SQLite backend, no cloud deps. macOS launchd or Linux systemd. AGPL-3.0 licensed.

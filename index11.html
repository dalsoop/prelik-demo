The Idea

Encrypted at entry.
Resolved at runtime.

The real secret exists only in process memory, for the duration of execution. Everything else — configs, git, logs, AI chat — sees only the amber reference.

  • AI agents see VK: references — work perfectly, zero risk
  • Key rotation happens per-node without restarts
  • Each node has its own DEK — lateral movement stops
  • Password from file only, never environment variable
Workflow

Four steps to preservation.

1

Scan

222+ patterns. AWS, GitHub, Stripe, private keys. Fully offline.

vk scan .env
2

Filter

Replace plaintext with AES-256-GCM encrypted VK: references.

vk filter .env
3

Wrap

Execute with auto-masking. Secrets can't leak to logs.

vk wrap ./deploy
4

Exec

Resolve at launch. In memory only, never on disk.

vk exec ./app

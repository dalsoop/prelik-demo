Workflow

Four steps. Zero plaintext.

1

Scan

Detect secrets across files with 222+ patterns. AWS keys, tokens, private keys, database URLs. Fully offline.

vk scan .env config/
2

Filter

Replace each detected secret with an AES-256-GCM encrypted reference. Plaintext destroyed immediately.

vk filter .env
3

Wrap

Run commands with automatic output masking. If a secret leaks to stdout, it's replaced before it reaches the terminal.

vk wrap ./deploy.sh
4

Exec

Resolve VK: tokens at runtime. Decrypted values injected into process memory — never written to disk.

vk exec ./my-app

Security Model

Three layers of defense.

◇

KEK

Key Encryption Key. Derived from password via scrypt. Never stored — derived on unlock.

◆

DEK

Data Encryption Key. 256-bit random, encrypted at rest with KEK. Rotatable per-node.

●

Ciphertext

Each secret encrypted with AES-256-GCM. Referenced as VK:{SCOPE}:{REF}. Unique nonce per entry.

⊘

No env vars

Passwords read from files only. No exposure via ps, /proc, or crash dumps.

⊕

Lateral isolation

Each node has its own DEK. Compromise one — the rest stay sealed.

⊙

Egress guard

Proxy intercepts HTTP. Shell hooks block direct curl, wget, and credential access.

Workflow

From plaintext to sealed in four steps

Every secret follows the same lifecycle. Detect, encrypt, reference, resolve.

1
Scan
Detect secrets in files and streams using built-in pattern engine. Fully offline.
vk scan .env
2
Filter
Replace each detected secret with an encrypted VK: reference token.
vk filter .env
3
Wrap
Execute commands with automatic output masking. Secrets never leak to logs.
vk wrap ./deploy.sh
4
Exec
Resolve VK: tokens at launch time. Injected into process memory, never written to disk.
vk exec ./app
Architecture

A hierarchy that isolates every node

KeyCenter decides policy. LocalVault holds ciphertext. Compromising one node doesn't expose the rest.

KC
KeyCenter :10181
Control plane. Manages node identity, DEK lifecycle, rotation policy, and the canonical secret registry. Single instance per deployment.
LV
LocalVault :10180
Node-local agent. Stores only encrypted ciphertext. Heartbeats every 5m. Cannot decrypt without KeyCenter policy approval.
PX
Proxy :18080
Optional egress enforcement. Monitors outbound traffic, detects plaintext leaks, blocks or rewrites with VK:TEMP: tokens.
CLI
CLI
Operator tool. scan, filter, wrap, exec, resolve, function management. Secure PTY wrapping with automatic masking.
Encryption Hierarchy
KEK
Key Encryption Key — derived from operator password. Never stored.
scrypt · N=32768 · r=8 · p=1
DEK
Data Encryption Key — 256-bit random. Encrypted at rest with KEK. Rotatable per-node.
AES-256-GCM · 12-byte nonce
Ciphertext
Each secret encrypted with current DEK. Referenced as VK:{SCOPE}:{REF}.
Per-secret unique nonce

Features

Everything you need, nothing you don't

◆

Zero plaintext at rest

Secrets encrypted at point of entry. Configs, logs, and env files contain only VK: references.

↺

Seamless rotation

KeyCenter schedules, each node re-encrypts independently. No restart, no downtime, no coordination.

⬡

Node isolation

Every node has its own DEK. Breach one, the rest stay sealed. Key hierarchy enforces lateral isolation.

⌗

222+ patterns

Detect AWS, GCP, GitHub, Stripe, private keys, JWTs, database URLs. Fully offline — no API calls needed.

▤

Egress enforcement

Optional proxy intercepts HTTP egress. Blocks or replaces plaintext secrets in outbound requests.

◎

Self-hosted

Runs on your infra. SQLite backend, no external deps. macOS launchd or Linux systemd. AGPL-3.0.

How It Works

Three steps to zero plaintext

VeilKey replaces secrets with encrypted references at the point of creation. Your code, configs, and logs never contain the real value.

Scan

Detect secrets across files, stdin, and CI pipelines using 222+ built-in patterns. No API needed — runs entirely local.

veilkey scan .env

Encrypt

Replace detected secrets with scoped VK: references. The original is encrypted with AES-256-GCM and stored in your vault.

veilkey filter .env

Resolve

At runtime, VK: tokens are resolved to real values — injected into the process without touching disk.

veilkey exec ./app
Architecture

Hub-and-spoke hierarchy

One KeyCenter controls the key lifecycle. LocalVault nodes hold only ciphertext. Compromising a node doesn't expose the fleet.

KeyCenter:10181
LocalVault A:10180
LocalVault B:10180
LocalVault Host:10180
CLI / Operator

KeyCenter control plane

Manages node identity, DEK lifecycle, rotation policy, and the canonical secret registry. Single instance per deployment. Never stores plaintext.

LocalVault node agent

Stores encrypted ciphertext locally. Reports heartbeats every 5 minutes. Executes rotation under KeyCenter policy. Cannot decrypt on its own.

Proxy egress enforcement

Optional layer that intercepts outbound traffic from wrapped workloads. Detects and blocks plaintext secrets in HTTP requests.

CLI operator tool

Scan, filter, wrap, exec, resolve. The operator interface for the entire secret lifecycle — from detection to rotation.

KEK
Key Encryption Key. Derived from operator password. Never stored.
scrypt(N=32768, r=8, p=1)
DEK
Data Encryption Key. 256-bit random. Encrypted at rest with KEK. Rotatable.
AES-256-GCM, 12-byte nonce
Ciphertext
Each secret encrypted with current DEK. Referenced as VK:{SCOPE}:{REF}.
Per-secret unique nonce
Features

Built for operators

⌘

No plaintext at rest

Secrets are encrypted the moment they enter the system. Config files, databases, and logs contain only VK: references.

↻

Zero-downtime rotation

KeyCenter schedules rotation. Each LocalVault re-encrypts independently. No service restart needed.

◈

Node isolation

Each node has its own DEK. Compromising one node doesn't expose secrets on other nodes.

⧖

Password-file only

KEK passwords are read from files, never environment variables. No exposure via ps, /proc, or crash dumps.

◉

222+ detection patterns

Scan for AWS keys, GitHub tokens, private keys, database URLs, and more. Runs entirely offline — no API needed.

⬡

Egress enforcement

Optional proxy intercepts outbound traffic and blocks leaked secrets. Veilroot shell guards block direct curl/wget.

▣

Self-hosted

Runs entirely on your infrastructure. No cloud dependencies, no phone-home, no vendor lock-in. AGPL-3.0 licensed.

⎔

Multi-platform

Native support for macOS (launchd) and Proxmox LXC (systemd). Single installer builds from source.

Quick Start

Running in minutes

Clone, build, run. One command to install everything from source.

Terminal — macOS
$ git clone https://github.com/veilkey/veilkey-selfhosted.git
$ cd veilkey-selfhosted

# Install everything (Go + build + deploy + init)
$ ./installer/scripts/install-mac.sh install
  [OK] Go installed
  [OK] veilkey-keycenter built
  [OK] veilkey-localvault built
  [OK] veilkey-cli built
  [OK] KeyCenter initialized (HKM root)
  [OK] LocalVault → KeyCenter connected

# Start services
$ ./installer/scripts/install-mac.sh start
  ✓ KeyCenter on :10181
  ✓ LocalVault on :10180

# Verify
$ veilkey status
  API: connected · Patterns: 222 loaded
Terminal — Proxmox LXC
# cd installer
# ./install.sh init

# Set passwords (never use env vars)
# echo -n 'your-password' > /etc/veilkey/keycenter.password
# chmod 600 /etc/veilkey/keycenter.password
# echo -n 'your-password' > /etc/veilkey/localvault.password
# chmod 600 /etc/veilkey/localvault.password

# Install and activate
# ./scripts/proxmox-lxc-allinone-install.sh --activate /
  ✓ KeyCenter installed and running
  ✓ LocalVault installed and running

# curl http://localhost:10181/health
  {"status":"ok"}
